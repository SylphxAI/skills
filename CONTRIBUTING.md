# Contributing

Sylphx Skills are written for agents, not as generic articles.

1. Add or update exactly one semantic owner under `skills/<id>/`.
2. Make the `description` identify when the package should and should not load.
3. Keep the procedure actionable, bounded, and independently useful.
4. Use original synthesis; do not copy third-party prose or include customer,
   credential, or incident data.
5. Update related packages only where their contract genuinely changes.
6. Run `npm run build:catalog` and `npm test`.

Do not introduce a second authoring root, meta-router, benchmark laboratory,
admission service, or generated copy of package bodies.
