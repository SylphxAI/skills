# Submit a Skill

Open a pull request with:

- `skills/<skill-name>/SKILL.md`;
- optional `references/`, `scripts/`, or `assets/`;
- an eval file under `evals/` for complex skills;
- a short PR body explaining what behavior improves.

## Naming

Use lowercase kebab-case. Keep names short and action-oriented.

Good:

- `interface-craft`
- `security-review`
- `pricing-page-critique`

Avoid:

- `ultimate-mega-ai-prompt`
- `helper`
- `misc-tools`

## Licensing

Default to original writing. If a skill is based on third-party material, include compatible license evidence and attribution in a reference file.
