# SEO Content Brief Patterns

## Brief components

| Component | Purpose |
| --- | --- |
| Intent | Know whether the searcher wants learn, compare, buy, troubleshoot, or navigate |
| Audience | Match sophistication and vocabulary |
| SERP expectation | Identify table, list, tool, template, review, tutorial, docs, or calculator expectation |
| Differentiation | Add original examples, data, templates, decision tables, or product workflow |
| Conversion path | Move readers to the next useful step without hijacking intent |
| Refresh plan | Keep facts, screenshots, pricing, and platform rules current |

## Rule IDs

- `seo-brief-1` — Start with search intent, not keyword volume.
- `seo-brief-2` — Cluster related queries around one useful page instead of creating near-duplicate pages.
- `seo-brief-3` — Define the page's original contribution before writing an outline.
- `seo-brief-4` — Use product screenshots, workflows, calculators, examples, or templates when they genuinely help the searcher.
- `seo-brief-5` — Map internal links by journey stage: learn, compare, implement, price, support.
- `seo-brief-6` — Include objections and comparison criteria for commercial investigation pages.
- `seo-brief-7` — Developer SEO needs accurate commands, API examples, version notes, and docs links.
- `seo-brief-8` — Localize and refresh pages where platform policies, pricing, or screenshots drift.
- `seo-brief-9` — Avoid doorway pages, copied outlines, hidden claims, and keyword stuffing.
- `seo-brief-10` — Measure qualified outcomes, not traffic alone.

## Decision table

| Intent | Page pattern | CTA |
| --- | --- | --- |
| Informational | Guide, checklist, explainer | Template, newsletter, related guide |
| Commercial | Comparison, alternatives, pricing guide | Demo, trial, calculator, case study |
| Transactional | Product/category page | Signup, checkout, install |
| Troubleshooting | How-to, support article | Support flow, product fix, docs |
| Developer | Tutorial, API docs, example app | CLI install, repo, docs, API key |

## Measurement

Track: impressions, rank bucket, qualified click-through, scroll depth, CTA click, signup/trial/install, assisted conversion, support deflection, freshness review date.
