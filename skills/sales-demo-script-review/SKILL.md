# Sales Demo Script Review

Use this skill to turn a recurring product-operations problem into a concrete, reviewable artifact instead of generic advice.

## Workflow

1. Identify buyer persona, pain, current workaround, desired outcome, sales stage, proof assets, and timebox.
2. Read `references/sales-demo-patterns.md`.
3. Build a demo around discovery, promised outcome, narrative, proof path, objection handling, and next commitment.
4. Tailor the script to buyer role instead of giving a generic feature tour.
5. Produce talk track, click path, timing, proof points, objections, and follow-up plan.

## When not to use

- Do not use for generic advice the base model already handles without this skill's specific artifact contract.

## Guardrails

- Do not invent customer logos, ROI, compliance, integrations, or roadmap commitments.
- Do not demo every feature; show the shortest credible path to the buyer outcome.
- Keep discovery questions before claims when pain is not proven.
- Separate product truth from sales hypothesis and roadmap possibility.

## Output format

```text
Demo context:
Buyer / pain / stage / timebox:

Demo script:
| Time | Segment | Talk track | Click/proof | Buyer check |
| --- | --- | --- | --- | --- |

Objection handling:
- <objection> -> <answer/evidence/follow-up>

Next step:
- <commitment, owner, deadline, proof>
```
