# Store Listing Optimization

Use this skill to make store pages communicate product value before users install or wishlist.

## Workflow

1. Identify channel, audience, product type, conversion goal, and current performance.
2. Read `references/store-listing-optimization-patterns.md`.
3. Review metadata, visual assets, proof, trust, policy fit, localization, and experiments.
4. Separate channel-specific constraints from universal conversion problems.
5. Produce prioritized listing changes and measurement plan.

## Guardrails

- Do not overclaim features, privacy, rankings, or compatibility.
- Do not optimize keywords at the expense of user understanding.
- Verify channel policy and asset specs before submission.

## Output format

```text
Channel:
Conversion goal:

Listing fixes:
- <rule id> <asset/field> -> <change>

Experiment plan:
- Hypothesis: <hypothesis>
  Asset: <title/screenshot/video/etc>
  Metric: <metric>

Policy/trust risks:
- <risk>
```
