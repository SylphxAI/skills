# Domain Selection Patterns

## Rule IDs

- `domain-select-1` — Optimize for say-spell-remember before cleverness.
- `domain-select-2` — Check acronym and homophone collisions in target markets (e.g., "OPC" industrial automation vs one-person company).
- `domain-select-3` — Prefer comprehension over trendy TLDs when trust matters to the buyer.
- `domain-select-4` — Compare total cost of ownership: renewal, transfer, privacy, and premium markup.
- `domain-select-5` — Run a lightweight SERP ambiguity check: does the name already mean something else?
- `domain-select-6` — Align domain with social handles and email domain strategy early.
- `domain-select-7` — Keep a backup acceptable name before announcing publicly.
- `domain-select-8` — Flag trademark and existing brand collisions for legal review, not casual dismissal.
- `domain-select-9` — Use country TLDs when local trust or regulation matters; use .com/.ai when global SaaS positioning matters.
- `domain-select-10` — Document redirect plan for common misspellings only when cost is justified.

## TLD decision table

| TLD | Best fit | Tradeoff | Example buyer signal |
| --- | --- | --- | --- |
| .com | Global default, highest trust | Often taken or expensive | Mainstream SaaS, consumer apps |
| .ai | AI-native positioning | Renewal premium, hype association | AI products, dev tools |
| .io | Developer and startup norm | Cost, geopolitical sensitivity for some enterprises | Dev tools, infra |
| .co | Startup alternative to .com | Confusion with .com | Early-stage brands |
| .app / .dev | Product and engineering clarity | HTTPS requirements, narrower feel | Apps, developer products |
| ccTLD | Local trust, regulation | Limits global expansion story | Regulated or local markets |

## Selection checklist

- [ ] Pronounceable on a phone call without spelling twice
- [ ] No unintended meaning in primary launch languages
- [ ] SERP check for dominant alternate meanings
- [ ] Trademark / existing brand screen completed (legal follow-up noted)
- [ ] Social handle availability checked on priority channels
- [ ] Email domain plan defined (Google Workspace, etc.)
- [ ] Renewal price acceptable for 3+ years
- [ ] Registrar supports transfer and 2FA
- [ ] Backup domain selected
- [ ] Redirect strategy defined for www/root and common typos if needed
