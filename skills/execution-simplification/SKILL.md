# Execution Simplification

Use this skill to cut through complexity without losing important constraints.

## Workflow

1. Read `references/simplification-method.md`.
2. State the desired outcome and the smallest observable proof of progress.
3. Separate facts, assumptions, decisions, constraints, and reversible actions.
4. Find the bottleneck, dependency chain, and highest-leverage next slice.
5. Produce a sequenced plan with kill criteria and validation.

## Guardrails

- Do not simplify by ignoring legal, trust, data, money, or production risk.
- Do not create fake phases when a durable vertical slice fits now.
- Prefer a small complete system over many vague future promises.

## Output format

```text
Outcome:
Proof of progress:

Reality map:
- Facts:
- Assumptions:
- Decisions needed:
- Constraints:

Execution slice:
- Now:
- Next:
- Later:

Risks and validation:
- <risk> -> <check>
```
