# Launch Demo Matrix

Each row is a short public demo prompt plus the artifact a good skill-shaped answer should produce.

| Skill | Demo prompt | Expected artifact |
| --- | --- | --- |
| `abuse-fraud-risk-review` | Review refund and promotion abuse risks for a mobile game with premium currency and referral rewards. | Starts from incentives/assets and proposes a control ladder with review and appeal.; Balances abuse prevention with privacy, false positives, support, and user trust. |
| `analytics-event-taxonomy` | Design an analytics taxonomy for a SaaS onboarding, checkout, subscription, refund, and support funnel. | Starts from decision questions and defines event names, properties, owners, identity, funnels, and QA.; Includes failure/guardrail metrics and privacy/cardinality constraints. |
| `app-store-distribution-readiness` | Prepare my iOS app and Android app for store launch. | Identifies target distribution channels first.; Covers metadata, privacy, payments, testing, review, release, and support.; Includes OS integration details and launch blockers. |
| `backup-restore-design` | Design backup and restore for a SaaS project management product. | Classifies data types and durability needs.; Defines RPO/RTO, restore states, conflicts, export, and support tooling.; Treats restore drills as required, not optional. |
| `changelog-and-release-notes` | Write release notes for a SaaS pricing and API change with migration and support implications. | Separates public notes, technical changelog, store notes, and support notes.; Calls out breaking changes, action required, support routes, and security/privacy constraints. |
| `customer-support-operations` | Design support operations for a subscription mobile app with refunds and restore-purchase issues. | Defines channels, issue taxonomy, triage, escalation, tooling, macros, and metrics.; Connects support patterns back to product fixes.; Handles refunds, backup/data loss, incidents, and abuse review with trust-preserving language. |
| `desktop-os-integration` | Create a macOS and Windows readiness checklist for a desktop AI utility with tray/menu bar, auto-update, backup, and local credentials. | Covers install, update, uninstall, startup, permissions, storage, credentials, files, notifications, and backup.; Separates macOS, Windows, and shared behavior. |
| `empty-state-and-loading-review` | Review empty, loading, offline, and error states for a SaaS dashboard. | Classifies the state type before prescribing UI.; Includes copy, action, accessibility, layout stability, recovery, and event checks. |
| `execution-simplification` | Turn this broad product roadmap into a simple execution plan with assumptions, risks, and validation. | Separates facts, assumptions, decisions, constraints, reversible actions, and proof.; Produces a sequenced plan with validation and avoids fake phases. |
| `game-economy-review` | Review a mobile puzzle game economy with daily rewards, rewarded ads, IAP bundles, refunds, and live events. | Maps currencies, sources, sinks, offers, events, refunds, and abuse risks.; Balances monetization with fairness, trust, and live-ops measurement. |
| `help-center-architecture` | Design help center architecture for a SaaS app with onboarding, billing, refunds, privacy, and troubleshooting. | Structures help by user problem/lifecycle and includes escalation, search, ownership, analytics, and product feedback.; Makes billing, refund, deletion, privacy, and account routes findable. |
| `incident-communication-playbook` | Draft incident communications for a payment outage that caused entitlement delays and support tickets. | Separates known facts, unknowns, mitigation, workaround, next update, support macro, and follow-up.; Handles payment/data/privacy/security incidents with higher review and precise wording. |
| `interface-craft` | Review this React form and make it feel more polished without changing the product direction. | Identifies concrete UI surfaces before suggesting details.; Prioritizes accessibility and reduced-motion constraints.; Uses rule IDs when reviewing. |
| `landing-page-critique` | Critique this SaaS landing page and propose higher-converting sections without making false claims. | Diagnoses audience, promise, proof, CTA, objections, and measurement.; Proposes experiments with hypotheses and metrics. |
| `launch-readiness-review` | Run a launch readiness review for a paid mobile app with subscriptions, App Store review, support, and promotion. | Gives go/conditional/no-go based on blockers and blast radius.; Checks product, monetization, distribution, support, analytics, trust, operations, promotion, and rollback. |
| `market-research-synthesis` | Research the market for a new Mac utility and synthesize what we should build. | Uses multiple source surfaces, not one competitor.; Separates table stakes from differentiation opportunities.; Produces original synthesis rather than copied competitor structure. |
| `mobile-app-product-systems` | A user bought premium currency, spent it, then got a refund. Should we ban them or ask them to buy again? | Classifies the mobile product surface before giving advice.; Defines entitlement, refund, restore, promotion, notification, and reward edge cases.; Separates entitlement revocation from account bans and uses an abuse policy ladder.; Designs promotions with objective, audience, eligibility, placement, fulfillment, measurement, and guardrails.; Rejects fake scarcity, manipulative streaks, and disruptive ads. |
| `mobile-first-ui-review` | Review this mobile onboarding and subscription flow for one-handed usability and trust. | Audits reachability, safe areas, inputs, loading, offline, settings, and permission timing.; Uses rule IDs and separates blockers from polish. |
| `motion-transition-review` | Review this modal and page transition system for subtle modern motion. | Classifies the state change before recommending animation.; Includes reduced-motion and interruptibility requirements. |
| `notification-strategy-review` | Design notification strategy for a mobile game with daily rewards, promos, and support-safe refund messages. | Builds a taxonomy by channel, consent, frequency, and user value.; Includes quiet hours, preferences, event schema, and negative-signal metrics. |
| `onboarding-activation-review` | Review onboarding for a SaaS app that asks for signup, import, team invite, and payment before value. | Defines activation as first value, not screen completion.; Maps friction, permission timing, first action, metrics, and experiment backlog. |
| `payment-platform-readiness` | Audit an iOS subscription payment flow with Apple Pay, IAP, restore purchases, refunds, and support tooling. | Separates catalog, checkout, receipt/webhook, entitlement, refund, support, and reconciliation.; Rejects client-only durable grants and includes idempotent ledger events. |
| `pricing-page-critique` | Critique this SaaS pricing page for conversion and trust. | Checks value metric, plan differentiation, risk reducers, cancellation, and measurement.; Rejects dark patterns and hidden material terms. |
| `privacy-and-data-retention-review` | Review privacy and retention for a SaaS app with analytics, billing, support tickets, backups, and account deletion. | Maps collection, use, storage, sharing, retention, deletion/export, backups, and disclosures.; Flags sensitive data, support access, third-party SDKs, and store form consistency. |
| `product-lifecycle-architect` | Plan the full product operating system for a mobile subscription app, from idea to support. | Classifies product type and business model before planning.; Covers planning, research, design, monetization, distribution, operations, growth, and support.; Identifies state machines, edge cases, metrics, and trust risks. |
| `product-positioning` | Sharpen positioning for a SaaS product that currently says AI productivity platform for everyone. | Identifies segment, job, alternative, category, differentiator, proof, and bad-fit boundary.; Rejects unsupported uniqueness and vague taglines. |
| `promotion-campaign-review` | Design a mobile app promotion for expired subscribers that increases win-back without coercive messaging. | Maps objective, audience, offer, eligibility, placement, fulfillment, metrics, and guardrails.; Rejects fake urgency and unclear promotion terms. |
| `refund-and-support-flow-review` | Design refund and support handling for a mobile game where users refund consumables after spending them. | Separates refund, entitlement, account action, support, and abuse review.; Rejects automatic bans and coercive repurchase threats. |
| `responsive-layout-review` | Audit this SaaS dashboard so it works from phone to wide desktop. | Uses content stress points rather than arbitrary device-only breakpoints.; Checks tables, navigation, forms, modals, hover, keyboard, and long content. |
| `saas-subscription-pricing` | Design pricing tiers for a B2B SaaS with usage-based costs and team collaboration. | Identifies buyer, value metric, cost driver, and expansion path.; Covers trial/freemium/tiers/usage/enterprise tradeoffs.; Includes cancellation, downgrade, renewal, metrics, and trust concerns. |
| `seo-content-brief` | Create an SEO content brief for a developer tool alternatives page that should convert without sounding spammy. | Starts from intent and original value, not keyword stuffing.; Includes outline, proof, internal links, CTA, freshness, and qualified metrics. |
| `skill-marketplace-creator` | Design a public repository for sharing AI agent skills. | Starts repository-first unless a heavier marketplace is justified.; Separates skills from MCP tools.; Includes validation, safety, license, and eval gates. |
| `startup-growth-review` | Audit this landing page for growth and conversion issues. | Separates diagnosis from assumptions.; Prioritizes actions by impact, confidence, speed, and strategic fit.; Rejects spammy or deceptive growth tactics. |
| `store-listing-optimization` | Optimize an App Store listing for a subscription utility with weak install conversion. | Separates channel assets and policies while focusing on conversion and trust.; Includes experiment plan and retained-user guardrails. |
| `subscription-entitlement-review` | Audit a SaaS subscription system with trials, downgrade, failed renewal, refund, and support adjustment paths. | Separates commerce state, ledger events, and derived entitlement state.; Includes state machine, event schema, support tooling, and user messaging. |
